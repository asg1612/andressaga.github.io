SSH sin password

La información de este documento se ha extraido de http://www.linuxproblem.org/art_9.html

La razón principal por la que realizo esta configuración es para tener scripts que puedan acceder, enviar o recibir ficheros de un equipo a otro sin tener que guardar contraseñas.

La configuración que hay que hacer es la siguiente:

  • En el equipo desde el que vamos hacer ssh: origen@usuario:~> ssh-keygen -t rsa Generating public/private rsa key pair. Enter file in which to save the key (/home/a/.ssh/id_rsa): Created directory '/home/usuario/.ssh'. Enter passphrase (empty for no passphrase): Enter same passphrase again: Your identification has been saved in /home/usuario/.ssh/id_rsa. Your public key has been saved in /home/usuario/.ssh/id_rsa.pub. The key fingerprint is: xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx:xx origen@usuario

  • Creamos si no existe el directorio remoto. origen@usuario:~> ssh b@B mkdir -p .ssh destino@usuario's password:

  • Añadimos la nueva clave al servidor:

origen@usuario:~> cat .ssh/id_rsa.pub | ssh b@B 'cat >> .ssh/authorized_keys' destino@usuario's password:

A note from one of our readers: Depending on your version of SSH you might also have to do the following changes:

Put the public key in .ssh/authorized_keys2 Change the permissions of .ssh to 700 Change the permissions of .ssh/authorized_keys2 to 640
